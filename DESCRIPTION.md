# Salesforce Experience Cloud PRM Sandbox

Salesforce Experience Cloud PRM Sandbox helps product managers, admins, architects, and QA testers preview partner portal experiences without repeatedly logging in and out with different test users.

Open a Salesforce Experience Cloud sandbox or staging portal, then use the extension popup to create partner personas such as Tier 1 Reseller, Referral Affiliate, MSP, distributor, or regional partner variants. Each persona can store partner type, tier, and region values detected from the connected Salesforce environment.

Use Single View to apply one persona context to the current tab. Use Split-Screen, Compare Three, or Compare Four to preview multiple partner personas side by side in the same browser tab. This makes it easier to compare visibility, page layout behavior, field differences, moderation rules, deal registration access, MDF visibility, and other partner-specific experiences before a portal release.

For best results, run comparison mode from the published Experience Cloud site URL, such as a `my.site.com` sandbox or staging domain. Salesforce Builder, Setup, and Lightning admin pages may block iframe rendering by policy, so the extension guides users back to the embeddable Experience Cloud site when needed.
